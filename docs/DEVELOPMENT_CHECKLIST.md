# 개발 체크리스트

> 최종 업데이트: 2025-12-30  
> 목적: 모든 개발 작업 시작 전 필수 확인 사항

---

## 🚨 개발 시작 전 필수 확인

### 1단계: 시스템 현황 확인 (필수)

**목적**: 중복 설치 방지, 기존 환경 활용

- [ ] **시스템 현황 문서 확인**
  - 파일: `docs/SYSTEM_STATUS.md`
  - 확인 사항: Python 경로, 패키지 설치 상태, 환경 설정

- [ ] **빠른 참조 가이드 확인**
  - 파일: `docs/QUICK_REFERENCE.md`
  - 확인 사항: 주요 명령어, 스크립트 경로

- [ ] **기존 구현 확인**
  - 목적: 중복 작업 방지
  - 방법: 코드베이스 검색으로 유사 기능 확인

---

### 2단계: 관련 문서 확인 (필수)

**목적**: 개발거버넌스 준수, 가이드라인 확인

- [ ] **개발거버넌스 문서 확인**
  - 파일: `docs/DEVELOPMENT_GOVERNANCE_GUIDE.md`
  - 확인 사항: 코딩 스타일, 모듈 구조, 계약 준수

- [ ] **관련 가이드라인 확인**
  - 코딩 스타일: `docs/CODING_STYLE_GUIDE.md`
  - 모듈 구조: `docs/MODULE_STRUCTURE_GUIDE.md`
  - 계약 준수: `docs/CONTRACT_COMPLIANCE_GUIDE.md`

- [ ] **관련 프로세스 문서 확인**
  - 파일: `docs/PROCESS_REVIEW_AND_STATUS.md`
  - 확인 사항: 기존 프로세스, 개선 사항

---

### 3단계: 가이드라인 검증 (필수)

**목적**: 코드 품질 보장, 가이드라인 준수

- [ ] **가이드라인 검증 실행**
  ```bash
  python scripts/validate_development_guidelines.py --src
  ```

- [ ] **검증 결과 확인**
  - 에러: 0개 (필수)
  - 경고: 가능한 한 최소화

- [ ] **검증 실패 시 수정**
  - 에러는 반드시 수정
  - 경고는 우선순위에 따라 수정

---

### 4단계: 중복 작업 확인 (필수)

**목적**: 불필요한 작업 방지, 토큰 효율 향상

- [ ] **유사 기능 검색**
  - 코드베이스에서 유사 기능 검색
  - 기존 구현 재사용 가능 여부 확인

- [ ] **문서 검색**
  - 관련 문서에서 유사 작업 확인
  - 기존 해결 방안 확인

- [ ] **체크리스트 확인**
  - `docs/GOVERNANCE_USAGE_AUDIT.md` - 활용 현황
  - `docs/REAL_KIWOOM_TEST_STATUS.md` - 테스트 상태

---

## 📋 개발 중 체크리스트

### 코딩 스타일 준수

- [ ] 타입 힌트 추가 (함수 파라미터, 반환값)
- [ ] Docstring 작성 (Args, Returns, Raises)
- [ ] 에러 처리 구현
- [ ] 로깅 추가 (필요 시)

### 계약 준수

- [ ] 인터페이스 계약 준수
- [ ] 데이터 형식 계약 준수
- [ ] 에러 처리 계약 준수

---

## 📋 개발 완료 후 체크리스트

### 검증

- [ ] 가이드라인 검증 통과
- [ ] 계약 검증 통과
- [ ] 단위 테스트 작성 (커버리지 80% 이상)
- [ ] 통합 테스트 작성

### 문서화

- [ ] 코드 주석 추가
- [ ] 관련 문서 업데이트
- [ ] 변경 사항 문서화

## 🚀 빠른 실행

### 자동 체크 스크립트

```bash
# 개발 시작 전 자동 체크
python scripts/start_development.py --task "작업 설명"
```

**기능**:
- 시스템 현황 자동 확인
- 관련 문서 자동 검색 및 표시
- 가이드라인 검증 자동 실행
- 중복 작업 확인

1. **문서 우선**: 개발 시작 전 반드시 관련 문서 확인

3. **검증 필수**: 가이드라인 검증 통과 후 커밋
4. **토큰 효율**: 필요한 문서만 참조
